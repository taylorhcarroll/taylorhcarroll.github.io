What I do

User Research

I put users first when designing applications. It all starts with defining the problem, competitor analysis, interviewing stakeholders and synthesizing all of this information into something easily communicated back to the client. Armed with this data, I can begin designing confident my decision making will always point back to the user.

Wireframing & Prototyping

I start my wireframing process with quick iterations on lo-fi mockups and get them into the hands of users as quickly as possible. This allows me to quickly try many different ideas until I get something that is intuitive and effective and based on real user feedback. This allows me to quickly prototype while reducing development time and budget waste.

User Testing & Hi-fi Mockups

Once I have something solid in lo-fi wireframes, I translate this to a hi-fidelity mockup like Figma. I build the mockup to simulate what the final product might look like and how the user interacts with it as closely as I am able to in this stage of the process to get further feedback. Discoveries can be made at each stage of the process and this allows me fine-tune the design before any code has been committed to Github.

Front End Development

I can build a responsive front end in vanilla javascript or a framework like React, and Im just as comfortable using CSS with flexbox as I am with front end frameworks like Material UI. I construct easily understood, commented code with the aim to scale up with as little refactoring as possible. I can take a project from Challenge Statement to a fully realized application.

Information Architecture

Information Architecture is more than just how to organize a website. It is how the information is interconnected in relation to a user's process. How you arrange the information at each stage of user's journey is critical to making best possible user experience. I tackle this by performing my own analysis by tagging entries in a set of data with tags and noting the patterns that emerge but also by asking a set of test users to arrange the data themselves.

APIs and Data

I have experience with Postman and have consumed Metro Government, Eventbrite, Zomato, and Ticketmaster APIs. I am very familiar with creating flat databases and displaying them in ERDs. I know when adding a join-table early on in development process can minimize future headaches.
